# sentences 修正スキル

ユーザーから「この jp + kana の組み合わせはこういう理由でNG」というケースを受け取り、次の3つを行う：

1. `src/lib/sentences.toml` 内の該当エントリを修正または削除する
2. `.claude/skills/generate-sentences/SKILL.md` の生成・検証プロンプトにそのNGパターンを追記する
3. 変更をコミットする

## ステップ 1: NG情報のヒアリング

ユーザーに以下をまとめて確認する（まだ情報が足りない場合のみ）：

1. **NG な jp と kana のペア**（複数あれば全部）
2. **NG の理由**（各ペアについて）
3. **修正方針**: 各エントリについて —
   - **削除**: そのエントリを sentences.toml から取り除く
   - **修正**: kana（または jp）を正しい値に直す（正しい値も教えてもらう）

ユーザーがすでに十分な情報を提示している場合はこのステップをスキップし、確認なしに次へ進む。

## ステップ 2: sentences.toml の修正

`src/lib/sentences.toml` を `Read` ツールで読み込み、対象エントリを特定する。対象エントリが見つからない場合はユーザーに確認して（jp / kana の正確な値を再確認）、ここで作業を中断する（ステップ 3・4 も実施しない）。

### 削除の場合

`Edit` ツールで該当の `[[sentences]]` ブロック（空行 + `[[sentences]]` + `jp = ...` + `kana = ...` の4行）をまるごと削除する。

ファイル先頭のエントリで空行がない場合は、`[[sentences]]` + `jp = ...` + `kana = ...` の3行と、続く区切り空行（次エントリとの間にある1行）もまとめて削除する。

### 修正の場合

`Edit` ツールで該当の `kana = "..."` 行（または `jp = "..."` 行）のみを正しい値に置き換える。同じ kana 値を持つ別エントリが存在しうる場合は `jp = ...` 行と合わせた2行を `old_string` に指定して一意性を確保する。

### 変更後

`Bash` で `npm run gen` を実行して `src/lib/generated/sentences.json` を再生成する。

## ステップ 3: generate-sentences スキルの更新

`.claude/skills/generate-sentences/SKILL.md` を `Read` ツールで読み込み、生成プロンプトと検証プロンプトの両方にNGパターンを追記する。

**Read 後、追記する前に既存内容との重複を確認する。**
- 同一の例が既に記載されている → スキップ（追記しない）
- 類似のルールはあるが例が不足している → 新規追記ではなく、既存項目に例を追加する形で `Edit` する
- 全く新しい観点 → 末尾または最も近い項目の直下に追記する

### 追記場所と形式

**生成プロンプト側（`## チェックリスト` セクション）**:

NGパターンの性質に応じて、既存の近いチェック項目の直下か末尾に追記する。
- 読み誤り系（複合語・促音便など）→ 読み関連の既存項目の直下
- 文字種・フォーマット系 → 対応する既存項目の直下
- 内容・意味系 → チェックリスト末尾

**検証プロンプト側（`## NG の例` テーブル）**:

テーブルの末尾に行を追記する：

```
| <jp> | <kana> | <NG理由（簡潔に）> |
```

判定基準（`## 判定基準` セクション）は、**既存の項目（例: 読み誤り・助詞の誤用・多読み漢字など）でカバーされない新しい種類のNG** である場合のみ追記する。既存項目で説明できるなら追記不要。

### 追記の方針

- **汎化する**: 特定の単語だけでなく「同種のミスのパターン」として書く。例: 「夜空→よるそら」の修正は「複合語を構成漢字の個別読みで繋げない」というルールとして記述する
- **簡潔に**: 既存のルールと重複しない範囲で最小限の追記にとどめる
- **具体例を添える**: ルールの説明には `例: X → Y（× Z）` 形式で添える

## ステップ 4: コミット

```bash
git add src/lib/sentences.toml src/lib/generated/sentences.json .claude/skills/generate-sentences/SKILL.md
git commit -m "fix: <変更内容を端的に表すメッセージ>"
```

コミットメッセージの body には修正したNG例と理由を箇条書きで含める。
